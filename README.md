## What is praxichnology?

Praxichnology is the study of *traces* (Greek *ichnos* — literally, footprint) produced through *situated activity* (Greek *praxis*) in *designed environments* — instrumented digital habitats including serious games, learning systems, and AI-agent workflows.

The foundational architecture is laid out in the v1.0 preprint:

> Loh, C. S. (2026). *Praxichnology: Naming the field that reads footprints in designed environments — from serious games to AI.* https://doi.org/10.5281/zenodo.20148907

For the etymology, the call-and-answer framing across Bekhterev and Simon as intellectual ancestors, and the field-vs-stance architecture, see the preprint at [`praxichnology/preprint`](https://github.com/praxichnology/preprint).

## Zenodo *and* GitHub: two platforms, two purposes

Praxichnology lives in two places. This is intentional, not redundant — they serve different scholarly functions, and educator-researchers unfamiliar with the convention often ask why both are needed.

| | **Zenodo** ([praxichnology community](https://zenodo.org/communities/praxichnology/)) | **GitHub** ([praxichnology](https://github.com/praxichnology)) |
|---|---|---|
| Role | Library | Workshop |
| Content state | Immutable per version (citation integrity) | Mutable (corrigenda, edits, iteration) |
| Identifiers | DOIs (permanent, resolver-backed) | Repository URLs (can change if renamed) |
| Discoverability | Indexed in DataCite, OpenAIRE, Google Scholar, and similar scholarly indexes | GitHub search, organic links |
| Use case | "Where do I cite it from?" | "Where does the project live and develop?" |

In short: **Zenodo is what you cite; GitHub is where the work happens.** A DOI from Zenodo guarantees that a reader following your citation in 2040 lands on the exact text you read, regardless of any subsequent edits.

For educator-researchers used to journal publishing, the analogy is: Zenodo is like the version-of-record on the journal's archival server; GitHub is like a working draft folder on your own machine. The preprint workflow simply gives you both citability *and* transparency without waiting for a journal queue.
